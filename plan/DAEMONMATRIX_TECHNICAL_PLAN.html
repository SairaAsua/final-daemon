Git Flow

Ramas permanentes

main — únicamente código que compila y corre. Protegida.

develop — integración continua (opcional para equipos grandes).

Ramas por área (vida ≤ 2 días)

Reglas de merge

  1. Cada 2 días, el lead de área abre PR a main.
  2. El PR debe pasar: tests + lint + review de ≥ 1 humano.
  3. Si un PR rompe main: revert inmediato + post-mortem async.
  4. Tags: v0.x.y-area-fecha en cada merge exitoso.
  5. Eko procesa PRs abiertos cada mañana: resume cambios, detecta conflictos, alerta si hay área sin merge ≥ 2 días.
